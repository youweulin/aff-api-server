# GoSavor 推薦策略

## 核心原則
用戶正在日本旅遊，掃描行為 = 即時需求信號。根據信號推薦「現在就能用」的商品。

## 用戶行為信號 → 推薦邏輯

### 1. 掃描類型判斷

| 掃描行為 | 代表什麼 | 推薦什麼 |
|---------|---------|---------|
| 掃菜單 | 正在餐廳附近 | 附近景點門票、美食體驗 |
| 掃收據（藥妝） | 在購物區 | 購物優惠券、附近景點 |
| 掃收據（餐廳） | 剛吃完飯 | 下午行程、甜點體驗 |
| 掃籤詩/告示 | 在景點/寺廟 | 附近體驗、和服租借 |
| 掃交通標誌 | 在移動中 | 交通券、一日遊 |

### 2. 地點判斷（從店名/GPS推斷城市區域）

| 地區 | 推薦重點 |
|------|---------|
| 東京新宿/渋谷 | teamLab、東京鐵塔、歌舞伎町體驗 |
| 東京淺草 | 和服租借、隅田川遊船、人力車 |
| 東京秋葉原 | 女僕咖啡、動漫體驗 |
| 大阪道頓堀 | 美食體驗、章魚燒課程 |
| 大阪 | 環球影城、大阪城 |
| 京都 | 和服租借、茶道體驗、嵐山竹林 |
| 奈良 | 東大寺門票、餵鹿體驗 |
| 福岡/博多 | 屋台體驗、太宰府 |
| 北海道 | 小樽運河、滑雪體驗 |
| 沖繩 | 美麗海水族館、潛水 |

### 3. 時間判斷

| 時間段 | 推薦什麼 |
|--------|---------|
| 早上 (6-11) | 一日遊（還來得及出發）、早市體驗 |
| 中午 (11-14) | 美食體驗、料理教室 |
| 下午 (14-17) | 景點門票、體驗活動 |
| 傍晚 (17-20) | 居酒屋體驗、夜景、屋形船 |
| 晚上 (20+) | 隔天的一日遊（預訂明天的） |

### 4. 掃描頻率判斷

| 行為模式 | 推薦策略 |
|---------|---------|
| 第一次掃描 | 交通券（可能剛到）、熱門景點 |
| 持續掃描多天 | 深度體驗（料理教室、文化體驗） |
| 同一區掃很多次 | 該區域的隱藏景點 |
| 掃很多收據 | 購物優惠券、免稅資訊 |

## 推薦優先級

```
Priority 1（必推）：
├── 🎫 當地景點門票（即買即用）
├── 🗓 明天的一日遊（晚上推）
└── 🚃 交通券（第一天推）

Priority 2（情境推）：
├── 🍽 美食體驗（掃菜單時）
├── 👘 文化體驗（掃寺廟/景點時）
└── 🛍 購物優惠（掃收據時）

Priority 3（補充）：
├── 📸 攝影景點推薦
└── 🎁 伴手禮推薦
```

## 推薦展示位置

| 位置 | 時機 | 形式 |
|------|------|------|
| 掃描等待中（3-5秒）| 每次掃描 | 全屏卡片，可跳過 |
| 翻譯結果下方 | 掃描完成 | 2-3 個小卡片 |
| 首頁最近掃描下方 | 開啟 App | 1-2 個推薦 |
| 日記頁面 | 回顧時 | 「當時附近的體驗」 |

## 轉換優化

| 策略 | 做法 |
|------|------|
| 即買即用 | 標示「今天可用」「電子票券」 |
| 距離感 | 標示「距你 500m」（用 GPS） |
| 折扣驅動 | 標示「比現場買便宜 20%」 |
| 社交證明 | 標示「1,234 位旅客購買」 |
| 時間壓力 | 「今日特價」「剩 3 張」 |
